# Zombie Event Shop

Темний Next.js-магазин для Minecraft Java 1.20.1 івенту "Зомбі проти людей". Вхід на івент безкоштовний, а набори й ресурси купуються за внутрішню валюту сайту — талери. Баланс можна поповнити через monobank або вручну в адмін-панелі. Покупки доступні 24/7, а видача відбувається після старту стріму та відкриття сервера. Проєкт не використовує офіційні логотипи Minecraft, Mojang або Microsoft.

## Стек

- Frontend: Next.js App Router, TypeScript, Tailwind CSS.
- Backend: Next.js API routes.
- DB: Prisma + SQLite для локального тесту.
- Payment: monobank acquiring API.
- Admin: пароль + HttpOnly session cookie.

## Запуск

1. Встановіть залежності:

```bash
npm install
```

2. Скопіюйте env:

```bash
cp .env.example .env
```

3. Створіть SQLite базу й засійте товари:

```bash
npm run db:push
npm run db:seed
```

4. Запустіть сайт:

```bash
npm run dev
```

Сайт відкриється на `http://localhost:3000`.

## Де вставити monobank token

Токен вставляється тільки у `.env` на сервері:

```env
MONOBANK_TOKEN="ваш_merchant_token"
MONOBANK_JAR_ID="your_jar_id_here"
MONOBANK_MOCK_MODE="false"
APP_URL="https://ваш-домен.example"
```

Не додавайте токен у `NEXT_PUBLIC_*`, компоненти React або будь-який frontend-файл.

`APP_URL` має бути публічним HTTPS-доменом, щоб monobank міг викликати webhook:

```text
https://ваш-домен.example/api/monobank/webhook
```

`MONOBANK_PUBLIC_KEY` можна залишити порожнім: код сам отримає й закешує public key через merchant API. Якщо ключ уже є, можна вставити його в env.

## Як протестувати оплату

Локально за замовчуванням увімкнений mock:

```env
MONOBANK_MOCK_MODE="true"
```

У цьому режимі кнопка поповнення балансу веде на локальний endpoint `/api/dev/mock-payment`, який позначає поповнення як `paid`, додає талери на баланс і перенаправляє на сторінку успіху. Це зручно для перевірки поповнень, checkout за балансом і адмін-панелі без реальної оплати.

Для реального тесту:

1. Отримайте merchant token monobank acquiring.
2. Встановіть `MONOBANK_MOCK_MODE="false"`.
3. Зробіть `APP_URL` публічним HTTPS URL, наприклад через ngrok або Cloudflare Tunnel.
4. Перезапустіть dev/prod сервер.
5. Поповніть баланс через `/top-up`.
6. Оплатіть інвойс на сторінці monobank.
7. Перевірте, що webhook додав талери на баланс у `/account` і показав транзакцію в `/admin`.

Для ручної перевірки webhook у dev можна тимчасово поставити:

```env
MONOBANK_SKIP_WEBHOOK_SIGNATURE="true"
```

Не використовуйте це в production.

## Адмін-панель

Адмін-панель доступна на `/admin`.

Посилання на адмінку приховане з навігації та футера. Відкривати її потрібно напряму.

Пароль задається в `.env`:

```env
ADMIN_PASSWORD="надійний-пароль"
ADMIN_SESSION_SECRET="довгий-випадковий-секрет"
USER_SESSION_SECRET="інший-довгий-випадковий-секрет"
```

Функції:

- перегляд усіх замовлень;
- фільтри `pending`, `paid`, `issued`, `failed`, `cancelled`;
- показ Minecraft-ніка, контакту, товарів, суми в талерах, статусу й дати;
- готові `/give` команди з підставленим ніком;
- кнопка "Позначити як видано" для замовлень, що очікують видачі;
- кнопка "Скасувати" для замовлень, які ще не видані;
- пошук гравця за ніком або email;
- перегляд балансу, поповнень, покупок і руху талерів;
- ручне додавання або списання талерів;
- перемикач статусу стріму для ручної видачі ресурсів;
- додавання, редагування й вимикання товарів.

Покупки доступні 24/7. Перемикач стріму не блокує замовлення, а лише показує адміну, що сервер відкритий для видачі після старту стріму.

## Категорії товарів

- Базові набори
- Шалкери виживання
- Шалкери бою
- Суперпредмети
- Супер ресурси

Квитків і косметики в каталозі немає: вхід безкоштовний, а косметичні привілеї не використовуються. Супер ресурси продаються варіантами x16 / x32 / x64, мінімальне замовлення ресурсів — від 16 шт.

## Prisma

SQLite використовується для тесту:

```env
DATABASE_URL="file:./dev.db"
```

Для переходу на PostgreSQL змініть provider у `prisma/schema.prisma`:

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

Після цього оновіть `DATABASE_URL`, запустіть міграції або `prisma db push`.

## Майбутня автоматична видача

Зараз видача ручна через адмін-панель і готові команди. Для RCON або окремого Minecraft-плагіна можна додати worker, який бере тільки замовлення зі статусом `pending`, виконує команди, а потім переводить їх у `issued`.
